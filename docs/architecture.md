## 核心算法

### Reddit Hot Ranking

热度排名采用 Reddit 经典算法：

```
hot_score = log10(|score|) + (timestamp / 45000)
```

**参数说明：**
- `score = upvotes - downvotes` - 净投票数
- `45000` = 12.5 小时（时间衰减常数）
- `timestamp` - 帖子发布时间戳

**算法特点：**
- 新帖和高赞帖都能获得合理排名
- 时间衰减确保老帖子逐渐下沉
- 对数函数使高票帖子优势递减

### Karma 计算

用户声望（Karma）计算公式：

```
karma = 帖子总点赞数 + 评论总点赞数
```

## Redis 缓存架构

### 数据结构设计

```
# 1. 热度排行榜 (Sorted Set)
hot:posts:global          # 全局热门帖子 ZSET
hot:posts:community:{id}  # 社区热门帖子 ZSET

# 2. 帖子详情缓存 (Hash)
post:detail:{id}          # 帖子完整详情 JSON (TTL: 10分钟)

# 3. 交互计数器 (Hash)
post:interactions:{id}    # 交互计数 (view_count, share_count, upvote_count, downvote_count)

# 4. 收藏数据 (Hash)
bookmarks:user:{id}       # 用户收藏列表 (post_id -> {folder, note, created_at})

# 5. 投票数据 (Hash)
votes:post:{id}           # 帖子投票数据
votes:comment:{id}        # 评论投票数据
```

### 热度更新流程

```
用户交互 (投票/浏览/分享)↓
Redis: 更新计数器 + 重新计算 hot_rank + 更新 ZSET
    ↓
异步任务: 每分钟批量同步到 PostgreSQL
```

### 交互权重配置

| 交互类型 | 权重 | 说明 |
|---------|------|------|
| 投票 | 10 | 权重最高，直接参与 score 计算 |
| 分享 | 5 | 中等权重，反映内容传播度 |
| 浏览 | 1 | 低权重，累计到热度 |

### 缓存策略

| 场景 | 策略 | TTL |
|------|------|-----|
| 热门列表 ZSET | 永久，实时更新 | 无 |
| 帖子详情 Hash | 缓存失效自动删除 | 10 分钟 |
| 交互计数器 | 永久，定时同步 | 无 |
| 帖子编辑/删除 | 立即失效缓存 | 立即 |
| 收藏数据 | 永久，实时更新 | 无 |

## 服务层架构

后端采用分层服务架构：

```
api/endpoints/     → 路由层（处理HTTP请求）
    ↓
core/services/     → 服务层（业务逻辑）
    ├── auth/      → 认证服务
    ├── content/   → 内容服务
    ├── admin/     → 管理服务
    └── infrastructure/ → 基础设施服务
    ↓
models/            → 数据层（ORM模型）
```

## 异步任务

系统使用异步任务处理定时操作：

| 任务 | 说明 |
|------|------|
| 数据同步 | 每分钟将 Redis 数据同步到 PostgreSQL |
| 统计任务 | 定期计算系统统计数据 |
| 热度更新 | 实时更新内容热度排名 |
